# 安全策略

## 报告问题

请通过 GitHub 仓库的 Security Advisory 私密报告漏洞。请勿在公开 Issue 中附加真实内部文档、用户路径或其他敏感信息。

## 运行安全边界

- 本地操作页面只监听 `127.0.0.1`。
- 运行时不使用模型 API、远程更新或命令桥接。
- 每次启动使用随机会话令牌保护转换接口。
- 输入会检查文件大小、DOCX 内部数量、解压体积、压缩比和路径。
- 工作文件保存于当前用户的临时目录，程序退出时删除。

## 发布要求

正式发布物需提供 SHA-256、SBOM 和平台签名。macOS 需完成 Developer ID 签名与公证，Windows 需使用可信 Authenticode 签名。
